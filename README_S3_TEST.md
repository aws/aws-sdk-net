# S3 Presigned POST SDK Testing Guide

This guide provides simple instructions for testing the S3 Presigned POST implementations across different AWS SDKs.

## Testing the .NET Implementation

```bash
# Navigate to the S3PostAotDemo directory
cd S3PostAotDemo

# Run the .NET demo
dotnet run
```

## Testing the JavaScript Implementation

```bash
# Navigate to the JavaScript test directory
cd js-s3-presigned-post-test

# Run the JavaScript demo
node index.js
```

## Running the Comparison Tool

```bash
# Navigate to the root directory (aws-sdk-net)
cd ..

# Run the comparison script
python compare_responses.py
```

The comparison script will detect matching response files from both implementations and generate a detailed report in `comparison_report.txt`.

## Notes

- Both implementations use the test cases defined in `s3-presigned-post-test-cases.json`
- Make sure you have AWS credentials configured in your environment
- The comparison tool automatically finds and compares response files generated by both implementations
